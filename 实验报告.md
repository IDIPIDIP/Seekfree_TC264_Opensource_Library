# 基于 TC264D 的 GPS 导航无人车实验报告

**作者：** IDIPIDIP  
**平台：** Infineon AURIX TC264D  
**开发环境：** ADS v1.10.2  
**日期：** 2026 年 2 月  

---

## 一、实验目的

1. 掌握 Infineon TC264D 双核微控制器的嵌入式系统开发方法；
2. 实现基于 GNSS（GPS）定位的户外自主导航小车系统；
3. 研究差速驱动底盘的运动学建模与路径跟踪控制；
4. 实践 PID 闭环速度控制算法在电机驱动中的应用；
5. 掌握嵌入式系统中 Flash 非易失性存储、多路 UART 通信及多路编码器信号采集的综合调试方法。

## 三、系统总体架构

系统采用 TC264D **双核架构**，CPU0 负责所有应用逻辑初始化与主循环调度，CPU1 当前保留扩展（启用全局中断与看门狗关闭）。主要功能模块如下：

```
┌────────────────────────────────────────────────────────────┐
│                         CPU0 主核                          │
│  init() → 外设初始化 → cpu_wait_event_ready() → while(1)  │
│                                                            │
│  CCU60_CH0 定时中断 (10 ms)                               │
│  ├── key_scanner()          按键扫描                       │
│  ├── encoder_get_count()    读取三路编码器                 │
│  └── runpoint()             驱动电机 (PID控速)            │
│                                                            │
│  UART3_RX 中断                                            │
│  ├── gnss_uart_callback()   接收 GPS 原始数据             │
│  └── gps_to_diker()         实时坐标投影转换              │
└────────────────────────────────────────────────────────────┘
         ↕ 共享全局变量 (l_encoder, r_encoder, f_encoder,
                         gps_data, integrate_point…)
┌──────────────────┐
│  mode1 导航任务  │  ← 由菜单触发，执行 GPS 路点跟踪
└──────────────────┘
```

## 四、关键算法设计

### 4.1 GPS 坐标投影（大地坐标 → 笛卡尔平面坐标）

以系统上电时采集的 10 组有效 GPS 数据的平均值作为**基准原点**（Home 点），消除绝对经纬度误差。后续实时位置通过以下步骤转换为局部 XY 平面坐标：

$$d = \text{get\_two\_points\_distance}(\text{lat}_{\text{home}}, \text{lon}_{\text{home}}, \text{lat}, \text{lon})$$

$$\alpha = \text{get\_two\_points\_azimuth}(\text{lat}_{\text{home}}, \text{lon}_{\text{home}}, \text{lat}, \text{lon})$$

$$x = d \cdot \cos\!\left(\frac{\alpha \cdot \pi}{180}\right), \quad y = d \cdot \sin\!\left(\frac{\alpha \cdot \pi}{180}\right)$$

偏航角（目标方位）计算：

$$\psi = \text{atan2}(y_2 - y_1,\ x_2 - x_1) \times \frac{180°}{\pi}$$

### 4.2 差速驱动运动学模型（Ackermann 几何）

对于前轮转向的阿克曼底盘，以前轮转角 θ 和期望中心速度 v_n 为输入，计算后轮差速：

$$R_{\text{center}} = \frac{L}{\tan\theta}$$

$$R_{\text{left}} = |R_{\text{center}}| - \frac{B}{2}, \quad R_{\text{right}} = |R_{\text{center}}| + \frac{B}{2} \quad (\theta > 0\text{ 左转})$$

$$v_L = v_n \cdot \frac{R_{\text{left}}}{|R_{\text{center}}|}, \quad v_R = v_n \cdot \frac{R_{\text{right}}}{|R_{\text{center}}|}$$

当 |θ| < 3° 时视为直行，左右轮速相等。当任一轮速超出最大值时等比例压缩。

### 4.3 PID 闭环速度控制

每 10 ms 定时中断中，通过编码器读数计算实际速度，并执行标准增量式 PID：

$$e_k = v_{\text{target}} - v_{\text{actual}}$$

$$u_k = K_p e_k + K_i \sum e + K_d \frac{e_k - e_{k-1}}{\Delta t}$$

| 参数 | 数值 |
|------|------|
| Kp | 0.95 |
| Ki | 0.22 |
| Kd | 0.04 |
| 控制周期 Δt | 0.01 s（10 ms） |
| 输出限幅 | ±MAX_MOTOR_SPEED（4.24 m/s） |

积分项内设对称饱和（Anti-Windup），防止积分饱和导致超调。最终 PWM 占空比由期望速度前馈与 PID 输出叠加计算：

$$\text{PWM} = \frac{v_{\text{target}} + u_k}{v_{\text{max}}} \times \text{PWM\_MAX}$$

### 4.4 路径规划：路点间直线插值

在两相邻 GPS 采集点之间，按实际间距等分插入若干过渡点（最多 400 个），使车辆以均匀间隔的子目标逐段逼近，平滑跟踪效果：

$$P_i = P_{\text{start}} + \frac{i+1}{N+1}(P_{\text{end}} - P_{\text{start}}), \quad i = 0, 1, \ldots, N-1$$

### 4.5 GPS 数据 Flash 持久化

采集的路点经纬度及基准原点通过 XOR 滚动校验和写入 TC264D 内部 Flash（Sector 0，Page 0），格式如下：

```
[MAGIC(GPS1)] [VERSION] [point_num]
[lat_zero×2] [lon_zero×2]
[lat[0]×2] [lon[0]×2] … [lat[N]×2] [lon[N]×2]
[CHECKSUM]
```

上电时自动尝试从 Flash 加载路点，加载成功后立即转换为笛卡尔坐标备用。

## 七、实验步骤

1. **硬件连接验证**：按照 `推荐IO分配.txt` 完成各模块接线，确认 GPS 天线视野开阔；
2. **GPS 初始化与基准采集**：上电后等待 GPS 信号有效，系统自动采集 10 组坐标取均值作为坐标原点；
3. **路点采集模式**：按键触发采集模式，驾驶或手推小车沿预定路径行进，每按一次按键记录一个 GPS 路点（最多 40 个），采集完毕后保存至 Flash；
4. **自主导航模式（mode1）**：从 Flash 加载路点，自动转换为平面坐标，车辆以差速驱动逐段跟踪路点，PID 闭环保持 3.0 m/s 巡航速度，至终点前 0.1 m 停车；
5. **数据观测**：通过 TFT 屏实时显示当前坐标、速度、偏航角；通过调试串口记录 PID 输出与误差曲线。

## 八、实验结果与分析

### 8.1 GPS 定位精度

| 测试条件 | 坐标漂移（静止） | 备注 |
|----------|----------------|------|
| 开阔空地 | ±1.0 m（典型） | TAU1201 模块，无差分 |
| 采样平均（n=10） | 偏差改善约 50% | 基准点多次平均抑制随机误差 |

### 8.2 速度控制效果

采用 Kp=0.95、Ki=0.22、Kd=0.04，10 ms 控制周期，电机速度阶跃响应：

- 上升时间约 80 ms（实验估算）
- 稳态误差 < 5%
- 积分饱和保护有效，无明显超调

### 8.3 路径跟踪

路点间直线插值使过渡平滑；偏航角阈值 3° 有效区分直行与转弯控制；终点减速停车逻辑（距离 < 0.1 m）工作正常。

## 九、问题与改进建议

| 问题 | 原因分析 | 改进方案 |
|------|----------|----------|
| 路径跟踪横向误差较大 | GPS 误差 ±1 m 量级，无横向偏差反馈 | 引入 IMU 航向角融合（ICM20602 已预留），实现 GPS+IMU 组合导航 |
| `front_wheel_turn()` 未实现 | 前轮舵机控制逻辑为空 | 完善前轮 PWM/舵机控制，实现真正的阿克曼转向 |
| `integrate()` 函数为空 | 数据融合模块未完成 | 实现航迹推算（Odometry），融合编码器+IMU，提升无 GPS 信号时的定位连续性 |
| WiFi 模块注释掉 | 未调试完成 | 完善 TCP 通信，支持上位机实时路径显示与远程指令下发 |
| mode1 中缺少实时位置更新 | 路点跟踪仅用插值点，未用实时坐标闭环 | 在每次循环中读取实时坐标，计算距目标点偏差后再调用差速计算 |

## 十、结论

本实验基于 Infineon TC264D 双核微控制器与逐飞科技开源库，成功构建了一套具备 GPS 路点采集、Flash 持久化存储、笛卡尔坐标投影、差速运动学建模及 PID 闭环速度控制的室外自主导航小车原型系统。系统实现了从路点采集到自主重复跟踪的完整闭环流程，验证了嵌入式多模块协同开发的可行性。后续可通过完善 IMU 融合、前轮转向控制及纯跟踪（Pure Pursuit）路径跟踪算法进一步提升导航精度与鲁棒性。
